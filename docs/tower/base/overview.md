# タワー定義・攻撃挙動 概要

## 目的・背景

タワーのデータ定義と、4種の攻撃タイプ別の挙動ロジックを実装する。タワーはScriptableObjectで定義し、コード変更なしにデータ追加・調整ができる設計にする。

## スコープ

### 作るもの

- TowerData（ScriptableObject）: 名前・ダメージ・CT・射程・サイズ・攻撃タイプ
- TowerBehaviour: グリッドに配置されたタワーのCTカウント・攻撃呼び出し
- 4種の攻撃タイプ実装: 照準型・自動追尾型・範囲型・設置型
- Projectile（弾丸）: 移動・当たり判定・消滅ロジック
- ObjectPool: 弾丸・設置オブジェクトのインスタンス管理

### 作らないもの

- グリッドへの配置処理（battle/preparation で対応）
- 合成システム（tower/synthesis で対応）
- タワーのアップグレード（economy で対応）

## 制約

- TowerData は ScriptableObject で定義し、Assets/Data/Towers/ 以下に配置する
- 弾丸はオブジェクトプールで管理する（最大プールサイズ: 50/タワー種）
- 攻撃タイプはインターフェース（IAttackBehaviour）で抽象化する

## 完了条件

- 初期タワー7種が TowerData として定義されている
- 各タワーが BattleScene で CT 通りに自動攻撃する
- 4種の攻撃タイプが仕様通りに動作する
- 弾丸がオブジェクトプールから生成・返却される

## 画面イメージ

（UI なし。ゲームフィールド上でのタワー動作のみ）

```
フィールド上での挙動例:

  [照準型タワー]  → マウス方向へ弾丸を発射
  [追尾型タワー]  → 最近の敵へ自動で発射
  [範囲型タワー]  → 自機周囲にエフェクト＋ダメージ
  [設置型タワー]  → 前方に地雷を設置
```
