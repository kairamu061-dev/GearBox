# タワー基盤 テストケース

## ユニットテスト

| ID | テスト名 | 入力 / 条件 | 期待結果 | ステータス |
|----|---------|------------|---------|-----------|
| U-01 | TowerData 読み込み | 蒸気砲の ScriptableObject を参照 | damage=30, cooldown=2.0, range=8.0 が正しく取得される | [ ] |
| U-02 | CT タイマー — 発射判定 | TowerBehaviour.Update を cooldown 秒分進める | IAttackBehaviour.Attack が1回呼ばれる | [ ] |
| U-03 | CT タイマー — 早期非発射 | Update を cooldown 未満で進める | Attack が呼ばれない | [ ] |
| U-04 | ObjectPool — 取得と返却 | Projectile を10回取得後に全返却 | プール内オブジェクト数が10に戻る | [ ] |
| U-05 | AimProvider — 座標変換 | スクリーン座標 (960, 540)（画面中央）を渡す | カメラ正面のワールド座標が返る | [ ] |
| U-06 | AimedAttack — 射線方向 | AimProvider が右方向を返す状態で Attack | 弾丸が右方向（velocity.x > 0）で飛ぶ | [ ] |
| U-07 | AutoAimAttack — 射程外 | 最近接敵が range 外にいる状態で Attack | 弾丸が生成されない | [ ] |
| U-08 | AutoAimAttack — 射程内 | 最近接敵が range 内にいる状態で Attack | 弾丸が敵方向へ飛ぶ | [ ] |
| U-09 | AreaAttack — 半径判定 | 範囲内に3体・範囲外に2体配置して Attack | 範囲内3体に TakeDamage が呼ばれ、範囲外2体には呼ばれない | [ ] |
| U-10 | PlacementAttack — 設置 | Attack 実行 | PlacementObject がタワー前方に生成される | [ ] |

## E2Eテスト

| ID | テスト名 | 前提条件 | 手順 | 期待結果 | ステータス |
|----|---------|---------|------|---------|-----------|
| E-01 | バトルでタワーが攻撃する | BattleScene に蒸気砲・追尾砲・範囲砲を配置、敵を近くにスポーン | 30秒バトルを実行 | 各タワーが攻撃タイプ通りに自動攻撃し敵のHPが減る | [ ] |

## ステータス凡例

- `[ ]` 未実施
- `[~]` 実施中
- `[x]` 合格
- `[!]` 不合格
