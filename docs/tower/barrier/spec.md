# バリア型攻撃 仕様

## 機能一覧

| # | 機能名 | 説明 |
|---|--------|------|
| 1 | シールド展開 | CTが0になった時点でシールドHPを展開し、被ダメを肩代わりする |
| 2 | シールド吸収 | 被ダメージをシールドHPから差し引き、シールドHP0で消滅する |
| 3 | 持続時間管理 | シールド展開から duration 秒経過で残HPに関わらず消滅する |
| 4 | CT再起動 | シールド消滅と同時にCTをリセットし、次の展開を待つ |
| 5 | 二重展開防止 | シールド展開中に再度CTが0になっても二重展開しない（CTのみリセット） |
| 6 | エフェクト制御 | シールド展開中・非展開中でビジュアルエフェクトを切り替える |

## タワー一覧

| タワー名 | シールドHP | CT(s) | 持続時間(s) | 概要 |
|---------|-----------|-------|------------|------|
| 蒸気装甲板 | 40 | 6.0 | 8.0 | 基本シールド |
| 圧力隔壁 | 80 | 10.0 | 12.0 | 高耐久・長CT |

## 動作仕様（BarrierBehaviour）

1. `TowerBehaviour` がCTを0にすると `IAttackBehaviour.Execute()` を呼ぶ
2. シールドが既に展開中の場合はCTのみリセットして終了する
3. `currentShieldHp = data.shieldHp` を設定してシールドを有効化する
4. シールド展開エフェクトを有効化する
5. `shieldDuration` のタイマーを開始する

### 被ダメ処理（TankHealth 側から呼ばれる）

```
shieldAbsorb = min(damage, currentShieldHp)
currentShieldHp -= shieldAbsorb
actualDamage = damage - shieldAbsorb
if currentShieldHp <= 0: DeactivateShield()
```

### シールド消滅（DeactivateShield）

1. `currentShieldHp = 0` にする
2. シールドエフェクトを無効化する
3. タイマーをキャンセルする
4. CTをリセットする（次の展開サイクル開始）

## TowerData 追加フィールド（バリア型専用）

| フィールド | 型 | 説明 |
|-----------|---|------|
| shieldHp | float | 1回の展開で吸収できる最大ダメージ量 |
| shieldDuration | float | シールドの最大持続時間（秒） |

## エラー / 異常ケース

| 条件 | 挙動 |
|------|------|
| シールド展開中に再度CTが0になった | CTのみリセット（二重展開なし） |
| 1回の攻撃でシールドHPを超えるダメージを受けた | シールドを0にして残ダメージを戦車HPへ通す |
| バリア型タワーを複数積んだ場合 | 各タワーが独立してシールドを管理する（合算なし） |

## 未対応ケース

- シールドの方向性（前面のみ保護など）
- 複数シールドの合算管理
- シールド残量のHUD表示
