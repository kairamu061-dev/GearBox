# タワー合成システム 概要

## 目的・背景

2つのタワーを素材として消費し、新しいタワーを生成する合成システムを実装する。合成によってより強力・特殊なタワーが生まれ、ビルド多様性とロールプレイを促進する。未知レシピの発見がランの探索動機にもなる。

## スコープ

### 作るもの

- SynthesisRecipe（ScriptableObject）: 素材A + 素材B → 結果タワー の定義
- 合成UI（改修ノード内）: タワー2つ選択 → 合成実行
- 既知・未知レシピの状態管理（RunManager 経由で永続）
- 合成結果の RunManager への反映（素材タワーを消費、結果タワーを追加）
- 設計図アイテム: 取得で未知レシピを既知にする

### 作らないもの

- 3種以上を使う合成
- タワー以外のアイテムとの合成

## 制約

- レシピは素材A・素材Bの順序を問わない（A+B = B+A として扱う）
- 未知レシピの結果は UI 上で「???」と表示し、試行するまで明かさない
- レシピは ScriptableObject で定義し、コード変更なしに追加できる

## 完了条件

- 合成UIで所持タワーから2つを選択して合成できる
- 既知レシピに一致する場合は結果タワーを生成し、素材2つを消費する
- 未知の組み合わせを試すと「合成失敗」として素材は消費されない
- 設計図を取得するとレシピが既知になり、UI に結果が表示される

## 画面イメージ

```
+------------------------------------------+
|  [ 合成 ]                                |
|                                          |
|  素材A        素材B          結果        |
|  [蒸気砲 ▼]  [機関銃 ▼]  → [蒸気ガトリング] |
|                                          |
|  ▶ 合成する                              |
|                                          |
|  既知レシピ                              |
|  ✓ 蒸気砲 + 機関銃 → 蒸気ガトリング     |
|  ✓ 散弾銃 + 炎放器 → 爆炎散弾砲         |
|  ? 電磁砲 + ??? → ???                   |
+------------------------------------------+
```
