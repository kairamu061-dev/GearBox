# タワー合成システム 仕様

## SynthesisUI の配置

`SynthesisUI` は **tower/synthesis/ が所有する共用 Prefab** であり、以下の2箇所で使用される：

| 使用箇所 | 配置方法 |
|---------|---------|
| PreparationScene の「合成」タブ | SynthesisPanel 内に Prefab をそのまま配置 |
| RefitScene の「合成」タブ | 同一 Prefab を参照（実装の追加なし） |

ロジック（`SynthesisManager`）も tower/synthesis/ が所有する。各シーンのコントローラは `SynthesisManager` を呼ぶだけでよい。

## 機能一覧

| # | 機能名 | 説明 |
|---|--------|------|
| 1 | レシピ定義 | SynthesisRecipe ScriptableObject で素材A・B・結果タワーを定義する |
| 2 | 合成UI（共用Prefab） | 所持タワーから2つを選択して合成を実行する。PreparationScene・RefitScene の両方で使用 |
| 3 | 合成実行 | 既知レシピに一致した場合、素材2つを消費して結果タワーを追加する |
| 4 | 未知レシピ処理 | 未知の組み合わせの場合、結果を「???」で表示し消費しない |
| 5 | レシピ開示 | 設計図アイテムを取得すると指定レシピが既知になる |
| 6 | 既知レシピ一覧 | 既知のレシピを一覧表示し、未知は「? + ? → ?」で伏せる |

## 操作方法

| 入力 | アクション |
|------|-----------|
| 素材Aドロップダウン選択 | 1つ目の素材タワーを選択する |
| 素材Bドロップダウン選択 | 2つ目の素材タワーを選択する |
| 「合成する」ボタン | 合成を実行する |

## ゲームプレイフロー

1. 改修ノードへ到着すると RefitScene が開く
2. 「合成」タブを選択する
3. 素材Aと素材Bのドロップダウンから所持タワーを選択する
4. 「合成する」を押す
5. レシピが既知かつ一致した場合: 素材2つを消費し、結果タワーを獲得する
6. 未知の組み合わせの場合: 「組み合わせ不明」と表示し、素材は消費されない
7. 既知だが素材不足の場合: ボタンが押せない状態にする

## 各ゲーム状態の詳細

### 素材未選択

- **表示内容**: 素材AとBのドロップダウン（空）、「合成する」ボタン（無効）
- **プレイヤー操作**: ドロップダウンでタワーを選択する
- **遷移条件**: 両方の素材が選択された
- **遷移先**: 素材選択済み状態

### 素材選択済み

- **表示内容**: 選択タワーのパラメータ、結果タワーのプレビュー（既知なら表示、未知なら「???」）
- **プレイヤー操作**: 「合成する」を押す、または選択を変更する
- **遷移条件**: 「合成する」が押された
- **遷移先**: 合成結果表示

### 合成結果表示

- **表示内容**: 成功なら結果タワー名・パラメータ。失敗なら「組み合わせ不明」
- **プレイヤー操作**: 「閉じる」を押して素材未選択状態へ戻る
- **遷移条件**: 「閉じる」ボタン
- **遷移先**: 素材未選択状態

## SynthesisRecipe ScriptableObject フィールド

| フィールド | 型 | 説明 |
|-----------|---|------|
| ingredientA | TowerData | 素材タワーA |
| ingredientB | TowerData | 素材タワーB |
| result | TowerData | 生成されるタワー |
| isKnownByDefault | bool | ゲーム開始時から既知か |

## 既知レシピの管理

- RunManager.KnownRecipes に既知の SynthesisRecipe の参照リストを持つ
- 設計図アイテム取得時に該当レシピを KnownRecipes へ追加する
- isKnownByDefault = true のレシピはラン開始時から KnownRecipes に含まれる

## エラー / 異常ケース

| 条件 | 挙動 |
|------|------|
| 素材Aと素材Bに同じタワーを選択した場合 | 「合成する」ボタンを無効にする（同一素材の合成は不可） |
| 所持が1個しかないタワーを両方の素材に選択 | ドロップダウン側でフィルタし選択不可にする |
| 既知レシピで合成成功後、素材タワーが0個になった | RunManager の所持リストから両タワーを削除して問題なし |

## 未対応ケース

- 3種以上のタワーを使った合成
- 合成の取り消し・アンドゥ
- 合成失敗時のスクラップ返却ペナルティ
