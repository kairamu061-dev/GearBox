# 自動追尾型攻撃 仕様

## 機能一覧

| # | 機能名 | 説明 |
|---|--------|------|
| 1 | 敵検出 | CT が 0 になった時点で Physics2D.OverlapCircle で射程内の敵を全取得する |
| 2 | ターゲット選択 | 取得した敵の中から HP が最低のものをターゲットにする |
| 3 | 弾丸発射 | ターゲット方向へ Projectile（tower/aimed/ と共用 Prefab）を発射する |
| 4 | 空振りスキップ | 射程内に敵がいない場合は CT をリセットせず、次フレームで再チェックする |
| 5 | CT未開始時の即時発射 | CT がない状態で敵が射程内に入った瞬間に発射し、そのタイミングから CT を開始する |
| 6 | ターゲット消失時の処理 | 弾丸飛行中にターゲットが破壊された場合、破壊地点まで飛行して着弾する |

## 動作仕様（AutoAimAttack）

### CT通常サイクル

1. `TowerBehaviour` が CT を 0 にすると `IAttackBehaviour.Execute()` を呼ぶ
2. `Physics2D.OverlapCircle` で射程内の敵を取得する
3. ターゲットがいれば HP 最低の敵を選び、Projectile を発射して CT をリセットする
4. ターゲットがいなければ何もしない（CT はリセットしない）

### CT未開始時（CT が進行していない状態）

5. 毎フレーム射程内を監視し、敵が射程内に入った瞬間に即時発射して CT を開始する

### ターゲット消失時

6. 弾丸飛行中にターゲットが破壊されると、弾丸はその破壊地点まで飛行して着弾する
7. 着弾時、範囲ダメージ有り → 範囲ダメージを発生させる。範囲ダメージなし → ダメージなし（空着弾）

## Projectile の扱い

- `Projectile` クラスおよび Prefab は tower/aimed/ で定義する
- `AutoAimAttack` は tower/aimed/ の ProjectilePool を参照して弾丸を取得する

## CT リセット方針

敵が射程内にいない場合、CT をリセット **しない**。
> **理由:** 弾丸1発分のクールダウンを空消費しないため。敵が射程に入った瞬間にすぐ発射できるよう CT を維持する。範囲型（tower/area）は「発動リズムを一定に保つ」設計のため常にリセットするが、自動追尾型は「弾を撃てるときだけ CT を消費する」設計として区別する。

## エラー / 異常ケース

| 条件 | 挙動 |
|------|------|
| 射程内に敵がいない | 攻撃をスキップし、CT を消費しない（リセットしない） |
| 全敵が死亡直後 | OverlapCircle の結果が空 → スキップ |

## 未対応ケース

- ホーミング弾（発射後にターゲットを追尾しない、直進のみ）
- 複数同時発射（散弾）
