# スクラップ経済システム テストケース

## ユニットテスト

| ID | テスト名 | 入力 / 条件 | 期待結果 | ステータス |
|----|---------|------------|---------|-----------|
| U-01 | ShopInventoryGenerator — タワー抽選 | GenerateInventory(seed=固定) を呼ぶ | タワーが出現率の範囲内で抽選される（毎回同じシードなら同じ結果） | [ ] |
| U-02 | ShopInventoryGenerator — 最大数 | GenerateInventory を呼ぶ | タワー・アイテムの合計が仕様の最大数を超えない | [ ] |
| U-03 | ShopSceneController — 残高不足で購入不可 | スクラップ30・価格50のアイテムで Purchase を呼ぶ | Purchase が false を返しスクラップが変動しない | [ ] |
| U-04 | ShopSceneController — 購入成功 | スクラップ100・価格50のタワーで Purchase を呼ぶ | スクラップが50減り所持タワーに追加される | [ ] |
| U-05 | ShopSceneController — sold フラグ | タワーを購入後に同じスロットを確認 | sold=true になり再購入できない | [ ] |
| U-06 | TowerUpgradeUI — Lv上限チェック | Lv3（上限）のタワーを選択 | 強化ボタンが無効化されている | [ ] |
| U-07 | RepairUI — 満タン時グレーアウト | HP=MaxHP の状態で RepairUI を開く | 修理ボタンが無効化されている | [ ] |
| U-08 | HatenaEventController — イベント抽選 | seed 固定で DrawEvent を呼ぶ | 出現率の範囲内でイベントが選ばれる | [ ] |

## インテグレーションテスト

| ID | テスト名 | 前提条件 | 手順 | 期待結果 | ステータス |
|----|---------|---------|------|---------|-----------|
| I-01 | ショップでタワー購入 → 所持タワー追加 | ShopScene を開く（スクラップ十分） | タワースロットの「購入」ボタンを押下 | RunManager.OwnedTowers にタワーが追加され、スクラップが減る | [ ] |
| I-02 | 残高不足でボタングレーアウト | スクラップ0でShopScene を開く | ShopScene の初期化 | 全ての購入ボタンがグレーアウトされている | [ ] |
| I-03 | 改修でタワー強化 | RefitScene を開く（スクラップ十分・Lv1タワーあり） | TowerUpgradeUI でタワーを選択して強化ボタン押下 | タワーが Lv2 になりコスト分スクラップが減る | [ ] |
| I-04 | 改修でタワー売却 | RefitScene を開く（タワーあり） | TowerSellUI でタワーを選択して売却ボタン押下 | タワーが所持一覧から削除され売却価格分スクラップが増える | [ ] |
| I-05 | ハテナイベント — スクラップ増加 | HatenaEventController でスクラップ増加イベントを発火 | イベント処理 | RunManager.CurrentScrap が指定量増加する | [ ] |

## E2Eテスト

| ID | テスト名 | 前提条件 | 手順 | 期待結果 | ステータス |
|----|---------|---------|------|---------|-----------|
| E-01 | ショップで設計図購入 → 合成可能に | 電磁砲+機関銃のレシピが未知の状態でショップへ | 設計図を購入する | レシピが解放され準備フェーズで合成できるようになる | [ ] |
| E-02 | 改修で修理 → バトルでHP確認 | HP=50/100 の状態で RefitScene へ | 修理量を選択して修理ボタン押下 → 次のバトルへ進む | BattleScene 開始時の HP が修理後の値（例：80/100）になっている | [ ] |
| E-03 | ハテナ — タワー選択肢 | ハテナノードを踏む | 「タワー3択」イベントが発生 | 3つのタワーが表示され選択したタワーが RunManager に追加される | [ ] |

## ステータス凡例

- `[ ]` 未実施
- `[~]` 実施中
- `[x]` 合格
- `[!]` 不合格
