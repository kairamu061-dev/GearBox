# ショップ テストケース

## ユニットテスト

| ID | テスト名 | 入力 / 条件 | 期待結果 | ステータス |
|----|---------|------------|---------|-----------|
| U-01 | ShopInventoryGenerator — 生成件数 | Generate() を呼ぶ | 在庫が 3〜5 件生成される | [ ] |
| U-02 | ShopInventoryGenerator — 同一シードで同一結果 | seed=42 で2回 Generate() | 両回とも同じ在庫内容 | [ ] |
| U-03 | ShopSceneController — 残高不足で購入不可 | スクラップ30・価格50 で OnBuyClicked | false を返しスクラップ変動なし | [ ] |
| U-04 | ShopSceneController — タワー購入成功 | スクラップ100・価格40 のタワーを購入 | スクラップが60になりタワーが TowerInventory に追加される | [ ] |
| U-05 | ShopSceneController — sold フラグ | タワーを購入後に同スロットを確認 | sold=true になり BuyButton が非アクティブ | [ ] |
| U-06 | ShopSceneController — 設計図購入 | 設計図アイテムを購入 | RunManager.UnlockRecipe が呼ばれる | [ ] |
| U-07 | ShopItemUI — グレーアウト | スクラップ0でシーン初期化 | 全 BuyButton が非インタラクティブになる | [ ] |

## インテグレーションテスト

| ID | テスト名 | 前提条件 | 手順 | 期待結果 | ステータス |
|----|---------|---------|------|---------|-----------|
| I-01 | ショップでタワー購入 → 所持タワー追加 | ShopScene を開く（スクラップ100） | タワースロットの「購入」ボタンを押下 | RunManager.TowerInventory にタワーが追加される | [ ] |
| I-02 | 再訪問時に在庫変化なし | ノードを訪問してショップを開く | 一度閉じて再度同じノードのショップを開く | 在庫が同じ（sold 状態も保持） | [ ] |

## E2Eテスト

| ID | テスト名 | 前提条件 | 手順 | 期待結果 | ステータス |
|----|---------|---------|------|---------|-----------|
| E-01 | 設計図購入 → 合成可能に | 電磁砲+機関銃のレシピが未知の状態でショップへ | 設計図を購入する | 準備フェーズで該当レシピが既知として表示される | [ ] |

## ステータス凡例

- `[ ]` 未実施
- `[~]` 実施中
- `[x]` 合格
- `[!]` 不合格
