# スクラップ経済システム 仕様

## 機能一覧

| # | 機能名 | 説明 |
|---|--------|------|
| 1 | スクラップ管理 | RunManager でスクラップ数を管理し、増減を処理する |
| 2 | ショップ購入 | ランダム生成された在庫からタワー・アイテムをスクラップで購入する |
| 3 | タワー強化 | スクラップを消費してタワーのパラメータをアップグレードする |
| 4 | HP修理 | スクラップを消費して戦車HPを回復する |
| 5 | グリッド拡張 | スクラップを消費して戦車グリッドを1行または1列増やす |
| 6 | タワー売却 | 不要タワーを解体してスクラップを獲得する |
| 7 | ハテナイベント | ランダムイベントでスクラップの増減・アイテム獲得が発生する |

## 操作方法

| 入力 | アクション |
|------|-----------|
| 「購入」ボタンをクリック | スクラップを消費してアイテムを購入する |
| 「強化」ボタンをクリック | スクラップを消費してタワーを強化する |
| 「修理」ボタンをクリック | スクラップを消費してHPを回復する |
| 「拡張」ボタンをクリック | スクラップを消費してグリッドを拡張する |
| 「売却」ボタンをクリック | タワーを解体してスクラップを獲得する |

## ショップ仕様

| 項目 | 内容 |
|------|------|
| 在庫数 | 3〜5種（ノード訪問時にランダム生成） |
| 在庫種別 | タワー（各種）、設計図（合成レシピ解放）、消耗品（HP回復など） |
| 価格 | TowerData / ItemData に定義した基本価格を使用 |
| 再生成 | 再訪問しても在庫は変化しない（ランを通じて固定） |

## 改修ノード仕様

### グリッド拡張

| 拡張内容 | コスト | 条件 |
|---------|--------|------|
| 列を1つ追加（横+1） | 80⚙ | 最大幅 5 まで |
| 行を1つ追加（縦+1） | 80⚙ | 最大高 5 まで |

### タワー売却

- 売却価格は購入価格の 50% または基本価格の 50%
- 強化済みタワーは強化コストの 25% も上乗せされる

| イベント名 | 内容 | 発生率 |

## 各ゲーム状態の詳細

### ショップ（ShopScene）

- **表示内容**: 在庫アイテム一覧（価格・パラメータ付き）、所持スクラップ数
- **プレイヤー操作**: 「購入」ボタンでアイテムを購入する
- **遷移条件**: 「閉じる」または「出発」ボタン
- **遷移先**: MapScene

### 改修（RefitScene）

- **表示内容**: タブ形式（強化/合成/修理/拡張/売却）、所持スクラップ数
- **プレイヤー操作**: タブ切り替え、各操作ボタン
- **遷移条件**: 「出発」ボタン
- **遷移先**: MapScene

## エラー / 異常ケース

| 条件 | 挙動 |
|------|------|
| スクラップ残高不足で購入しようとした | ボタンを無効化（残高不足の場合はグレーアウト） |
| 戦車HPが満タンの状態で修理 | 修理ボタンをグレーアウト |
| グリッドが最大サイズ（5×5）で拡張 | 拡張ボタンをグレーアウト |
| タワー強化が最大レベルの場合 | 強化ボタンをグレーアウト |

## 未対応ケース

- 複数通貨（スクラップ以外の希少通貨）
- オークション・競売形式のショップ
- スクラップ上限の設定
