# ハテナノード テストケース

## ユニットテスト

| ID | テスト名 | 入力 / 条件 | 期待結果 | ステータス |
|----|---------|------------|---------|-----------|
| U-01 | TriggerEvent — ウェイト抽選 | seed 固定で TriggerEvent を10回 | 各イベントの出現率がウェイトに近い分布になる | [ ] |
| U-02 | ScrapGain — 加算範囲 | ScrapGain イベントを処理（scrapMin=30, scrapMax=80） | スクラップが 30〜80 の範囲で増加する | [ ] |
| U-03 | ScrapLoss — 0止め | Scrap=10 で ScrapLoss（最大50減少）イベントを処理 | Scrap=0 で止まる（マイナスにならない） | [ ] |
| U-04 | BlueprintDiscover — 未知なしフォールバック | 全レシピが既知の状態で BlueprintDiscover | 「何もない」として処理される | [ ] |
| U-05 | RandomUpgrade — Lv3のみフォールバック | Lv3タワーのみ所持の状態で RandomUpgrade | 「何もない」として処理される | [ ] |
| U-06 | FullHeal — HP全回復 | HP=30/100 で FullHeal イベントを処理 | HP=100/100 になる | [ ] |

## インテグレーションテスト

| ID | テスト名 | 前提条件 | 手順 | 期待結果 | ステータス |
|----|---------|---------|------|---------|-----------|
| I-01 | TowerChoice — UI 表示と選択 | ハテナノードを踏む（TowerChoice イベント確定） | オーバーレイでタワー3種が表示されて1つ選択 | 選択したタワーが TowerInventory に追加される | [ ] |
| I-02 | オーバーレイの表示・非表示 | MapScene が表示中 | ハテナノードを踏んでイベント処理後に閉じる | オーバーレイが表示されて閉じたあと MapScene が正常に操作できる | [ ] |

## E2Eテスト

| ID | テスト名 | 前提条件 | 手順 | 期待結果 | ステータス |
|----|---------|---------|------|---------|-----------|
| E-01 | ハテナ踏んでレシピ解放 → 合成 | 電磁砲+機関銃のレシピが未知の状態 | ハテナで「設計図発見」イベントを踏む → 準備フェーズで合成を試みる | レシピが既知になり合成できる | [ ] |

## ステータス凡例

- `[ ]` 未実施
- `[~]` 実施中
- `[x]` 合格
- `[!]` 不合格
