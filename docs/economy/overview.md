# スクラップ経済システム 概要

## 目的・背景

スクラップを通貨とした経済システムを実装する。ショップ・改修ノードでのスクラップ消費がランの成長を担い、バトルでの回収量がリスクリワードのバランスを生む。

## スコープ

### 作るもの

- スクラップ数の RunManager 管理（増減・参照）
- ショップ（ShopScene）: タワー・アイテムの購入
- 改修ノード（RefitScene）: タワー強化・グリッド拡張・HP修理・タワー売却
- ハテナノードのランダムイベント実装（スクラップ報酬・ペナルティを含むもの）

### 作らないもの

- タワー合成の処理（tower/synthesis で対応）
- スクラップのドロップ・回収処理（enemy / battle/combat で対応）

## 制約

- スクラップは RunManager.Scrap（int）で管理する
- 残高不足のアイテムは UI で購入不可状態にする（マイナスにならない）
- ショップの在庫はノードごとにランダム生成し、再訪問しても変化しない

## 完了条件

- ショップでタワーを購入するとスクラップが消費され、タワーが所持に追加される
- 改修でタワー強化・修理・グリッド拡張が実行できる
- スクラップ残高が常に HUD に表示される
