## システム横断テスト

| ID | テスト名 | 前提条件 | 手順 | 期待結果 | ステータス |
|----|---------|---------|------|---------|-----------|
| S-01 | 準備 → 戦闘 → マップ の一連フロー | ラン開始済みで MapScene を表示 | バトルノード選択 → タワー配置 → 出撃 → バトルクリア | MapScene に戻りノードが訪問済みになる | [ ] |
| S-02 | HP と RunManager の整合性 | バトル開始時 HP=80 | バトルで敵弾を受けて撤退 | MapScene / 次の準備フェーズで HP=80−ダメージ量 が維持される | [ ] |
| S-03 | スクラップと RunManager の整合性 | バトル開始時スクラップ=100 | バトルで50スクラップ回収 | バトル終了後のマップ・ショップでスクラップが150になっている | [ ] |

## ステータス凡例

- `[ ]` 未実施
- `[~]` 実施中
- `[x]` 合格
- `[!]` 不合格
