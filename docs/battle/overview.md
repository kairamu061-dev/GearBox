# バトルシステム 概要

## 目的・背景

ゲームのコアメカニクスであるバトルを実装する。バトルはノード選択後に発生し、「準備フェーズ」と「戦闘フェーズ」の2段階で構成される。プレイヤーは戦術的なタワー配置と戦場での操縦を組み合わせてクリアを目指す。

## スコープ

### 作るもの

- 準備フェーズ（タワーグリッド配置UI）
- 戦闘フェーズ（戦車操作・タワー自動攻撃・目的地到達判定）
- バトルクリア条件（旗到達 or ボス撃破）
- ゲームオーバー条件（戦車HP0）
- バトル結果のRunManagerへの反映（スクラップ増減・HP更新）

### 作らないもの

- タワーそのものの実装（tower 機能エリアで対応）
- 敵そのものの実装（enemy 機能エリアで対応）
- スクラップの使用（economy 機能エリアで対応）

## 制約

- 準備フェーズと戦闘フェーズは独立して実装・テスト可能にする
- 戦闘フィールドはトップダウン2Dビュー
- 戦車の移動は Unity Physics 2D（Rigidbody2D）で実装する
- 照準はマウスカーソル位置をワールド座標に変換して使用する

## 完了条件

- 準備フェーズでタワーをグリッドに配置して出撃できる
- 戦闘フェーズでWASD移動・マウス照準が機能する
- 配置済みタワーが自動攻撃する
- 旗に触れるとバトルクリアになる
- HP0でゲームオーバー画面に遷移する
