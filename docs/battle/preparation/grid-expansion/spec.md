# グリッド拡張UI 仕様

## 機能一覧

| # | 機能名 | 説明 |
|---|--------|------|
| 1 | グリッドサイズ表示 | 現在のグリッドサイズ（例: 3×3）をテキストで表示する |
| 2 | スクラップ残高表示 | 現在の所持スクラップ数を表示する |
| 3 | 列追加ボタン | スクラップ 80⚙ を消費して列を1つ追加する |
| 4 | 行追加ボタン | スクラップ 80⚙ を消費して行を1つ追加する |
| 5 | ボタン有効/無効制御 | 残高不足またはグリッド上限（5×5）に達した場合はボタンをグレーアウトする |

## 操作フロー

1. 「列を追加」または「行を追加」ボタンをクリックする
2. 残高チェック（Scrap ≥ 80）かつ上限チェック（GridSize < 5）を通過する
3. `RunManager.SpendScrap(80)` を呼ぶ
4. `RunManager.ExpandGrid(addColumn: true/false)` を呼ぶ
5. 表示を更新する（サイズテキスト・ボタン状態）

## ボタン無効化条件

| ボタン | 無効化条件 |
|--------|-----------|
| 列を追加 | Scrap < 80 OR GridSize.x ≥ 5 |
| 行を追加 | Scrap < 80 OR GridSize.y ≥ 5 |

## 拡張コスト

| 操作 | コスト |
|------|--------|
| 列を追加（+1 列） | 80⚙ |
| 行を追加（+1 行） | 80⚙ |

## エラー / 異常ケース

| 条件 | 挙動 |
|------|------|
| スクラップ不足でボタンを押す | ボタンが無効化されているため押せない |
| グリッドが (5,5) でボタンを押す | ボタンが無効化されているため押せない |

## 未対応ケース

- 行・列の削除（縮小）
- 拡張コストの動的変化
