# マップ・ノードシステム 仕様

## 機能一覧

| # | 機能名 | 説明 |
|---|--------|------|
| 1 | マップ生成 | ランごとに分岐付きノードマップを手続き生成する |
| 2 | ノード表示 | 各ノードを種別アイコン付きで表示し、接続線を描画する |
| 3 | ノード選択 | 現在地から到達可能なノードのみ選択できる |
| 4 | 状態管理 | 訪問済み・未訪問・現在地をノードごとに管理する |
| 5 | ノード遷移 | 選択したノード種別に応じたシーンまたはイベントへ遷移する |
| 6 | エリアクリア判定 | ボスノード撃破後に次エリアへ進む |

## 操作方法

| 入力 | アクション |
|------|-----------|
| ノードをクリック | 到達可能なノードを選択して遷移 |
| （将来）マップをドラッグ | マップのパン移動 |

## ゲームプレイフロー

1. ランが始まるとマップが生成され、MapScene が表示される
2. スタートノードから到達可能なノード（隣接する次層のノード）がハイライトされる
3. プレイヤーがハイライト中のノードをクリックする
4. ノード種別に応じたシーンへ遷移する
5. イベント完了後、MapScene に戻る（現在地が選択したノードに更新される）
6. ボスノードを選択して撃破するとエリアクリア、次エリアのマップへ遷移する

## 各ゲーム状態の詳細

### MapScene（マップ閲覧・選択）

- **表示内容**: ノードグラフ、ノードアイコン、接続線、現在地マーカー、エリア番号
- **プレイヤー操作**: 到達可能ノードをクリックして選択
- **遷移条件**: ノードが選択された
- **遷移先**: 選択ノードの種別に対応するシーン（下記ノード種別参照）

### ノード種別と遷移先

| 種別 | アイコン | 遷移先 | 説明 |
|------|---------|--------|------|
| バトル | ⚔️ | PreparationScene → BattleScene | 通常戦闘 |
| ハテナ | ❓ | ランダムイベントUI（MapScene 上に表示） | ランダムイベント |
| ショップ | 🛒 | ShopScene | アイテム購入 |
| 改修 | 🔧 | RefitScene | タワー強化・合成・修理 |
| ボス | 💀 | PreparationScene → BattleScene（ボスフラグあり） | ボス戦 |

### ノードの状態

| 状態 | 表示 | 説明 |
|------|------|------|
| 未訪問・到達不可 | グレーアウト | まだ到達できないノード |
| 未訪問・到達可能 | ハイライト（点滅） | 現在選択できるノード |
| 訪問済み | 暗くなったアイコン | 既に通過したノード |
| 現在地 | 戦車アイコン表示 | プレイヤーの現在位置 |

## マップ生成パラメータ

| パラメータ | 初期値 | 説明 |
|-----------|--------|------|
| 層数 | 6 | スタートを除いたノード層の数（最終層はボス固定） |
| 各層ノード数 | 1〜3 | 層ごとのノード数をランダムで決定 |
| 分岐本数 | 最大2 | 1ノードから出る接続線の最大本数 |
| ノード種別の出現率 | バトル:50% ハテナ:20% ショップ:15% 改修:15% | 通常層でのランダム抽選率 |
| ボス層 | 固定（最終層） | 常に1ノードでボス種別 |

## 勝利 / 敗北 / 終了条件

| 種別 | 条件 | 結果 |
|------|------|------|
| エリアクリア | ボスノードのバトルに勝利 | 次エリアのマップへ遷移 |
| ゲームオーバー | バトル中に戦車のHPが0 | GameOverSceneへ遷移、ランリセット |

## エラー / 異常ケース

| 条件 | 挙動 |
|------|------|
| マップ生成時に孤立したノードができる | 生成後に全ノードがスタートから到達可能かチェックし、再生成する |
| バトル中にアプリを終了した | 次回起動時にそのノードのバトルからやり直し（セーブポイントはノード選択直前） |

## 未対応ケース

- エリアをまたいだマップの通し表示
- マップの手動編集・カスタムシード指定
- ノードに訪問時のアニメーション演出（スプライン移動など）は将来対応
