# マップ生成 仕様

## 機能一覧

| # | 機能名 | 説明 |
|---|--------|------|
| 1 | MapNode 定義 | ノード1件のデータ構造（ID・種別・座標・接続先・状態） |
| 2 | MapGraph 定義 | ノードリストとクエリAPI（到達可能ノード取得など） |
| 3 | マップ生成 | 層ごとにノード数・種別を抽選し分岐グラフを生成する |
| 4 | 到達可能性保証 | 生成後に孤立ノードが存在しないか検証し、失敗なら再生成する |
| 5 | 表示座標計算 | 各ノードの UI 表示用正規化座標（0〜1）を自動計算する |

## データ構造仕様

### NodeType

| 値 | 説明 |
|----|------|
| Battle | 通常戦闘ノード |
| Mystery | ハテナイベントノード |
| Shop | ショップノード |
| Refit | 改修ノード |
| Boss | ボス戦ノード（最終層固定） |

### NodeState

| 値 | 説明 |
|----|------|
| Unvisited | 未訪問・到達不可 |
| Reachable | 未訪問・到達可能（現在地から隣接） |
| Current | 現在地 |
| Visited | 訪問済み |

### MapNode フィールド

| フィールド | 型 | 説明 |
|-----------|---|------|
| id | int | ノードの一意ID（0始まり） |
| type | NodeType | ノード種別 |
| position | Vector2 | 表示用正規化座標（0〜1） |
| nextNodeIds | List\<int\> | 接続先ノードのIDリスト |
| state | NodeState | 現在の状態 |

### MapGraph API

| メソッド | 戻り値 | 説明 |
|---------|--------|------|
| GetNode(int id) | MapNode | IDからノードを取得 |
| GetReachableNodes() | List\<MapNode\> | currentNodeId の接続先ノード一覧 |

## 生成パラメータ

| パラメータ | 値 | 説明 |
|-----------|---|------|
| 層数 | 6 | スタートを除いた層の数（最終層はボス固定） |
| 各層ノード数 | 1〜3（ランダム） | 層ごとのノード数 |
| 分岐本数 | 最大2 | 1ノードから出る接続の最大数 |
| ノード種別出現率（通常層） | バトル 50% / ハテナ 20% / ショップ 15% / 改修 15% | 最終層はボス固定 |
| 第1層の種別 | Battle 固定 | ランの最初に選択するノードは必ずバトル |

## 生成アルゴリズム概要

1. スタートノード（id=0, state=Current）を作成
2. 第1層は全ノードを Battle 固定で生成する（ランダム抽選を行わない）
3. 第2層以降（最終層を除く）でノード数（1〜3）をランダム決定し、種別を抽選してノードを生成する
4. 前層の各ノードから次層のノードへ最大2本の接続を張る（全ノードが少なくとも1つの接続先を持つよう保証）
5. 最終層は1ノードでBoss固定
6. 全ノードの `position` をグリッド上の正規化座標として計算する
7. 到達可能性チェック（BFS/DFS）を実施し、孤立があれば再生成する

## エラー / 異常ケース

| 条件 | 挙動 |
|------|------|
| 孤立ノードが生成された | 全体を再生成する（最大10回リトライ） |
| リトライ上限到達 | 最小構成（直線マップ）にフォールバックする |

### 直線マップの定義（フォールバック）

| 項目 | 値 |
|------|---|
| 構造 | 各層に1ノードのみ・すべて直線接続（1本） |
| ノード種別 | 通常層はすべて Battle / 最終層は Boss |
| 層数 | 通常通り6層 |

## 未対応ケース

- カスタムシード指定
- エリアをまたいだマップの連結表示
