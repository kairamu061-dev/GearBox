# マップ・ノードシステム 概要

## 目的・背景

スレイザスパイヤ式のローグライクマップを実装する。プレイヤーはノードを選びながら進み、毎回異なるルートと体験を得られる。マップの分岐とノード多様性がランの戦略性と繰り返しプレイの動機を生む。

## スコープ

### 作るもの

- 分岐付きマップの生成・表示
- ノード種別の定義と遷移ロジック（バトル・ハテナ・ショップ・改修・ボス）
- ノード選択UI（到達可能ノードのみ選択可）
- エリアのクリア判定（ボスノード撃破後、次エリアへ）
- ランの進行状態管理（現在ノード・訪問済みノード）

### 作らないもの

- マップのスクロール演出（初期は静的表示）
- マップのセーブ・ロード（RunManager に委譲）
- ノード内のイベント実装（各機能エリアで対応）

## 制約

- マップはランごとに手続き的に生成する
- ノード数・分岐数はエリアごとにパラメータで調整可能にする
- 初期エリア: 縦5〜8層、各層1〜3ノード、分岐最大2本

## 完了条件

- マップ画面でノードが表示され、到達可能なノードを選択して遷移できる
- 訪問済みノードが視覚的に区別される
- ボスノードを撃破すると次エリアへ進める
- ランの進行状態がシーン遷移をまたいで保持される
