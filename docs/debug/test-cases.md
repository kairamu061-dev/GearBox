# デバッグシステム テストケース

## インテグレーションテスト

| ID | テスト名 | 前提条件 | 手順 | 期待結果 | ステータス |
|----|---------|---------|------|---------|-----------|
| I-01 | ログが Unity コンソールに出力される | エディタ再生中 | `Log.Info("test", LogTag.Debug)` を呼ぶ | `[HH:mm:ss.fff][INFO][Debug] test` 形式でコンソールに表示される | [ ] |
| I-02 | Unity ログがプールに追加される | エディタ再生中 | `Debug.Log("unity")` を呼ぶ | `GameLogger.EntryCount` が増加している | [ ] |
| I-03 | ファイル出力が行われる | ゲーム起動・ログが1件以上ある | デバッグメニューから「ログをファイルに出力」を押す | プロジェクトルートに `debug_log_*.txt` が生成される | [ ] |
| I-04 | ファイルの内容が正しい | I-03 の続き | 生成されたファイルを開く | タイムスタンプ・レベル・タグ・メッセージが時系列順に記載されている | [ ] |
| I-05 | ログクリアが動作する | ログが1件以上ある | デバッグメニューから「ログをクリア」を押す | `GameLogger.EntryCount` が 0 になる | [ ] |
| I-06 | デバッグメニューが全シーンで開く | ゲーム起動後 | LogoScene / MapScene / BattleScene それぞれで右上をタップ | デバッグメニューが開く | [ ] |
| I-07 | デバッグメニューが入力をブロックする | デバッグメニュー展開中 | メニュー外のゲームUI（ノードボタンなど）をタップ | ゲームUIが反応しない | [ ] |

## E2Eテスト

| ID | テスト名 | 前提条件 | 手順 | 期待結果 | ステータス |
|----|---------|---------|------|---------|-----------|
| E-01 | リリースビルドにデバッグコードが含まれない | Development Build のチェックを外してビルド | ビルドして実行する | `[DebugMenu]` GameObject が存在しない。`Log.Info` の呼び出しが除去されている | [ ] |

## ステータス凡例

- `[ ]` 未実施
- `[~]` 実施中
- `[x]` 合格
- `[!]` 不合格
