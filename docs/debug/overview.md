# デバッグシステム 概要

## 目的・背景

開発中の不具合調査・動作確認を効率化するために、ゲーム共通のロギングシステムとデバッグメニューを用意する。リリースビルドには一切含まれず、エディタ・開発ビルドでのみ動作する。

## スコープ

### 作るもの

- 構造化ログ出力クラス（`Log`）
- ログプールとファイル出力（`GameLogger`）
- ログタグ定義（`LogTag`）
- ゲーム中いつでも開けるデバッグメニュー（`DebugMenuController`）
- バグチケット管理（`Debug/tickets/`）

### 作らないもの

- ネットワーク越しのリモートロギング
- クラッシュレポートサービス連携
- ログの暗号化・難読化

## 制約

- エディタ（`UNITY_EDITOR`）および Development Build（`DEVELOPMENT_BUILD`）でのみ動作する
- リリースビルドではログ呼び出し自体がコンパイル時に除去される
- `DebugMenuController` はリリースビルドのバイナリに含まれない

## 完了条件

- `Log.Info("msg", LogTag.Battle)` の形でログが出力できる
- バトル中にデバッグメニューを開いてログファイルを出力できる
- リリースビルドでコンパイルエラーが出ない
