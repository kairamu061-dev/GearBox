# デバッグシステム 概要

## 目的・背景

開発中の不具合調査・動作確認を効率化するために、ゲーム共通のロギングシステムとデバッグメニューを用意する。
リリースビルドには一切含まれず、エディタ・開発ビルドでのみ動作する。

## 構成

| コンポーネント | 役割 |
|--------------|------|
| **Log**（静的クラス） | 構造化ログを Unity コンソールに出力する呼び出し口 |
| **GameLogger**（静的クラス） | ログエントリをプールし、ファイル出力を提供する |
| **LogTag**（enum） | ログに付与するタグの定義 |
| **DebugMenuController** | ゲーム中いつでも開けるデバッグUI |
| **バグチケット** | 発見した不具合を管理するMarkdownファイル群 |

## リリースビルドからの除外方針

| 手法 | 対象 |
|------|------|
| `[Conditional("UNITY_EDITOR"), Conditional("DEVELOPMENT_BUILD")]` | `Log.Info` 等のメソッド呼び出しをビルド時に自動削除 |
| `#if UNITY_EDITOR \|\| DEVELOPMENT_BUILD` | `DebugMenuController` クラス全体・`GameLogger` の初期化処理 |

Development Build でチェックを外したリリースビルドでは `Log.Info(...)` の呼び出し自体がコンパイル時に除去されるため、実行時のオーバーヘッドはゼロ。

## スコープ外

- ネットワーク越しのリモートロギング
- クラッシュレポートサービス連携
- ログの暗号化・難読化
